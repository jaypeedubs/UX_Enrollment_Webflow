Auth guard: Protected — redirect to /login if not signed in

── VARIABLES ──
  application      object    (from loadApplication request)
  notifications    list      (from loadNotifications request)
  notifDrawerOpen  boolean   default false
  unreadCount      number    default 0

── REQUESTS ──
  loadApplication
    Type   : Supabase — Get rows — table: applications
    Filter : applicant_id = auth.uid()
    Select : id, status, submitted_at, updated_at, programs(name)
    Limit  : 1
    Order  : created_at desc
    Run    : on page load

  loadNotifications
    Type   : Supabase — Get rows — table: notifications
    Filter : applicant_id = auth.uid()
    Order  : created_at desc
    Run    : on page load

    Type   : Supabase — Update rows — table: notifications
    Filter : applicant_id = auth.uid(), read = false
    Data   : { read: true }
    Run    : when notification drawer opens

  withdrawApplication
    Type   : Supabase — Update rows — table: applications
    Filter : id = @application.id
    Data   : { status: "withdrawn" }
    Then   : POST /rest/v1/application_events
             body: { application_id: @application.id, event_type: "withdrawn" }
    Then   : trigger loadApplication

  [wized=notif-drawer-close]    Click → set notifDrawerOpen = false

  [wized=notif-item-msg]        Loop on notifications; Text → @item.message
  [wized=notif-item-time]       Loop (same); Text → @item.created_at (format: "MMM D, YYYY")

  [wized=app-program-name]      Text → @application.programs.name
  [wized=app-next-action-msg]   Text → see Next-Action map below
  [wized=app-next-action-link]  href → see Next-Action map below
  [wized=view-status-link]      Visibility: show when application exists
  [wized=withdraw-btn]          Click → confirm dialog → trigger withdrawApplication

  [wized=start-application-link] Visibility: show when no application exists

PAGE 3 — /apply
Auth guard: Protected — redirect to /login if not signed in
On load: if application exists and status != draft, redirect to /dashboard
----------------------------------------------------------------

── VARIABLES ──
  applicationId    text      (from existing draft or newly created)
  programId        text      (selected program)
  currentSection   number    default 1   (1 | 2 | 3)
  programLocked    boolean   default false
  nameLocked       boolean   default false
  cvFileName       text      default ""
  cvUploaded       boolean   default false
  programQuestions list      (from selectedProgram.program_questions)

── REQUESTS ──
  loadPrograms
    Type   : Supabase — Get rows — table: programs
    Filter : status = "active"
    Select : id, name, deadline, program_questions
    Run    : on page load

  loadDraft
    Type   : Supabase — Get rows — table: applications
    Filter : applicant_id = auth.uid(), status = "draft"
    Select : id, program_id, status, cv_url, locked_fields, programs(name)
    Limit  : 1
    Run    : on page load
    On success with data: pre-fill form fields, set applicationId, set locks

  saveDraft
    Type   : Supabase — Upsert row — table: applications
    Body: {
      id          : @applicationId (if exists) or omit for auto-generate,
      applicant_id: @Wized.user.id,
      program_id  : @programId,
      status      : "draft",
      locked_fields: { program: true, first_name: true, last_name: true }
    }
    Update user metadata: first_name, last_name via Wized.auth.updateUser()
    On success: set applicationId from response; insert draft_saved event:
      POST /rest/v1/application_events
        body: { application_id: @applicationId, event_type: "draft_saved" }

  uploadCV
    Type   : Supabase Storage — Upload file
    Bucket : cvs
    Path   : @Wized.user.id + "/" + filename
    File   : @[wized=cv-file-input].files[0]
    On success: PATCH applications SET cv_url = storage_path WHERE id = @applicationId

  submitApplication
    Type   : Supabase — Update rows — table: applications
    Filter : id = @applicationId, status = "draft"
    Data   : { status: "submitted", submitted_at: new Date().toISOString(),
               locked_fields: { all: true } }
    Then   : POST /rest/v1/application_events
             body: { application_id: @applicationId, event_type: "submitted" }
    On success → redirect to /dashboard

── ELEMENT BINDINGS ──
  [wized=program-select]
    Options → loop on loadPrograms.data; value=id, label=name
    On change → set programId; load program_questions from selected program
    Disabled when programLocked == true

  [wized=program-locked]        Visibility: show when programLocked == true
  [wized=applicant-first-name]  Pre-fill: @Wized.user.user_metadata.first_name
                                Disabled when nameLocked == true
  [wized=first-name-locked]     Visibility: show when nameLocked == true
  [wized=applicant-last-name]   Pre-fill: @Wized.user.user_metadata.last_name
                                Disabled when nameLocked == true
  [wized=last-name-locked]      Visibility: show when nameLocked == true
  [wized=applicant-email]       Pre-fill: @Wized.user.email; always disabled (read-only)

  [wized=save-draft-btn]        Click → trigger saveDraft; show form-draft-status briefly
  [wized=next-section-1-btn]    Click → trigger saveDraft; set currentSection = 2
  [wized=form-draft-status]     Visibility: show 3s after successful saveDraft

  [wized=questions-loading]     Visibility: show while loadPrograms pending
  [wized=questions-empty]       Visibility: show when programQuestions.length == 0

  PROGRAM QUESTIONS LOOP — requires a repeating div in Webflow.

PAGE 5 — /enrollment-confirmation
Auth guard: Protected — redirect to /login if not signed in
On load: if status is not "accepted" or "enrollment_confirmed", redirect /dashboard
----------------------------------------------------------------

── VARIABLES ──
  application    object
  checkoutError  text    default ""

── REQUESTS ──
  loadApplication
    Type   : Supabase — Get rows — table: applications
    Filter : applicant_id = auth.uid(), status IN (accepted, enrollment_confirmed)
    Select : id, status, programs(name, price_cents)
    Limit  : 1
    Run    : on page load

  createCheckout
    Type   : HTTP POST (Edge Function — needs user JWT in Authorization header)
    URL    : https://xvweanlqcbgbiyxqhwux.supabase.co/functions/v1/create-checkout-session
    Headers: Authorization: Bearer @Wized.user.access_token
             Content-Type: application/json
    Body   : { application_id: @application.id }
    On success → window.location.href = response.url
    On error   → set checkoutError = response.error; show enroll-error-msg

── ELEMENT BINDINGS ──
  [wized=enroll-loading]         Visibility: show while loadApplication pending

  [data-w-id=...d465]  (program name span)   Text → @application.programs.name
  [data-w-id=...d46a]  (tuition span)        Text → "$" + (@application.programs.price_cents / 100).toFixed(2)
  [data-w-id=...d46f]  (status badge)        Text → @application.status == "enrollment_confirmed"
                                                       ? "Enrollment Confirmed" : "Accepted"

  ADD wized attributes to these three spans in Webflow:
    wized="enroll-program-name"
    wized="enroll-tuition"
    wized="enroll-status-badge"

  [wized=enroll-error-msg]       Text → @checkoutError
                                 Visibility: show when checkoutError is not empty

  [wized=enroll-already-enrolled] Visibility: show when application.status == "enrolled"
                                 (redirect away from page if this is somehow reached)

  [wized=enroll-processing]      Visibility: show while createCheckout pending

  [wized=confirm-enrollment-btn]
    Click → show enroll-processing; trigger createCheckout
    Visibility: show when status IN (accepted, enrollment_confirmed)
                AND createCheckout.status != "loading"

── REDIRECT GUARD (on page load) ──
  After loadApplication:
  - If data is empty (no accepted/enrollment_confirmed application) → redirect /dashboard
  - If application.status == "enrolled" → redirect /dashboard?payment=success
